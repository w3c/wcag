Creating Dialogs With The HTML dialog Element

ID:

Technology: html

Type: Technique

When to Use

HTML

Description

Site designers often want to create modal dialogs to get users to focus on a task-related activity outside of the rest of the page's content. Although this can be accomplished using ARIA to create a modal dialog, the HTML dialog element can also be used. As well as meeting the first rule of ARIA, the HTML dialog element offers several advantages to its ARIA equivalent, with the browser handling these features:

  • focus management into the newly-opened modal dialog;
  • focus management back onto the triggering element (assuming the trigger is still on the page) when the modal dialog is closed;
  • trapping keyboard focus inside the modal dialog;
  • with no extra code, the underlying page is hidden from assistive technology when the modal dialog is open;
  • the use of the Escape key to close the modal dialog.

This technique uses the HTML dialog element rather than the ARIA equivalent.

Examples

A dialog to sign up to a mailing list

This is an example of using a modal dialog element to show a mailing-list sign-up form to a user. The main part of the page contains a button element that, when activated, triggers the appearance of the modal dialog. The button uses the aria-controls property to create a programmatic connection between the button and the dialog; the aria-haspopup property, set to dialog, to tell assistive technology that the button controls a dialog, and the type attribute to tell the browser that this isn't a submit button.

When the modal dialog is opened, the browser will hide all of the content on the underlying page from assistive technology. For example: a screen reader will not announce any of the content. Additionally, focusing with the keyboard is constrained to the the dialog element's focusable content and the browser's controls. The browser takes care of focus management for us by putting the focus onto the first focusable element in the dialog's DOM, which in this demo is the h1 element because it has a tabindex="-1" attribute, making it focusable by scripts. Note that, although the dialog's close button is visibly before the h1, it is the last item in the dialog's DOM. If the button was first, it would receive focus when dialog was opened, which could result in the dialog being closed by mistake if the button was accidentally activated.

The HTML

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width,
     initial-scale=1, shrink-to-fit=no">
    <title>Turbo Encabulator News</title>
  </head>
  <body>
    <main>
      <h1>All The News About Turbo Encabulators</h1>
      <button aria-controls="mailing-list-dialog" aria-haspopup="dialog"
       class="open-modal" type="button">
       Sign up to our mailing list!
      </button>
    </main>
    <dialog aria-labelledby="dialog-heading" id="mailing-list-dialog">
      <h1 id="dialog-heading" tabindex="-1">Sign up to our mailing list</h1>
      <form>
        <p class="req-note">All form fields are required.</p>
        <div>
          <label for="fname">First Name</label>
          <input aria-required="true" autocomplete="given-name" id="fname" type="text">
        </div>
        <div>
          <label for="lname">Last Name</label>
          <input aria-required="true" autocomplete="family-name" id="lname" type="text">
        </div>
        <div>
          <label for="email">Email address</label>
          <input aria-required="true" autocomplete="email" id="email" type="text">
        </div>
        <button class="sign-up" type="submit">Sign up</button>
      </form>
      <button aria-label="close" class="close-modal" type="button">&times;</button>
    </dialog>
  </body>
</html>

The JavaScript

The script toggles the display of the dialog. The HTML dialog element can be opened using two different commands: show() (for non-modal dialogs), and showModal() (for modal dialogs).

document.addEventListener("DOMContentLoaded", function(e){
  const d = document.querySelector("dialog");
  const btnOpen = document.querySelector(".open-modal");
  const btnClose = document.querySelector(".close-modal");

  btnOpen.addEventListener("click", function(){
    d.showModal();
  }, false);

  btnClose.addEventListener("click", function(){
    d.close();
  }, false);
});

Tests

Procedure

  1. Find the components on the page that trigger modal dialog elements.
  2. Check that each dialog can be opened using the keyboard to activate the triggering element (for example: activate a button by pressing the Enter key or Space Bar.
  3. When the dialog is opened, check that focus is placed inside it.
  4. While the dialog is open, check that focus is trapped inside the dialog.
  5. When the dialog is closed, check that focus is placed back onto the triggering element, if that is still on the page.

Expected Results

  • Checks #2, #3, #4 and #5 are true.
