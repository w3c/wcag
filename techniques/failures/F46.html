Failure of Success Criterion 1.3.1 due to using structural elements or attributes in layout tables that are still exposed as data tables

ID: F46

Technology: failures

Type: Failure

When to Use

HTML

Description

The objective of this technique is to describe a failure that occurs when a table that is used only for layout is still exposed as a data table and includes structural elements or attributes. This is a failure because it exposes structural (or semantic) table markup in content only intended for presentation. The intent of the HTML table element is to present tabular data.

Assistive technologies use the structure of an HTML table to present data to the user in a logical and structured manner – for example, by announcing the total number of rows and columns, and announcing the content of header cells associated with the current data cell. This type of information is redundant and confusing in cases where a table structure has been used purely for layout purposes.

Although WCAG 2 does not prohibit the use of layout tables, CSS-based layouts are recommended in order to retain the defined semantic meaning of the HTML table elements and to conform to the coding practice of separating presentation from content.

Examples

Here is a simple example that uses a table to layout content in a three column format. The navigation bar is in the left column, the main content in the middle column, and an additional sidebar is on the right. At the top is a page title. The example marks the page title as th.

<table>
  <tr>
    <th colspan=3>Page Title</th>
  </tr>
  <tr>
    <td><div>navigation content</div></td>
    <td><div>main content</div></td>
    <td><div>right sidebar content</div></td>
  </tr>
  <tr>
    <td colspan=3>footer</td>
  </tr>
</table>

Tests

Procedure

For each table used only to visually lay out elements within the page:

  1. Using browser developer tools, check if the table is exposed as a data table
  2. Check if the table uses structural elements (such as th and caption) or attributes (such as headers and scope)

Expected Results

  • If checks #1 and #2 are tue, then this failure condition applies and the content fails the Success Criterion.
