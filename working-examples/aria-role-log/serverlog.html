Using role="log" with server log

This simple script demonstrates how a log file can have new content surfaced to ATs using the aria log role. The aria-relevant value of "additions" is specified so that entries removed due to log size limitations are ignored (however this is strictly for demonstration purposes since "additions" is the implicit default value). Pressing the Reboot server button causes a time-stamped log entry to be added. For each activation of the button, only the changes will be surfaced by the screen reader due to the implicit aria-atomic="false" value.
